1Ô∏è‚É£ Mapping Malaria Cases with sf & ggplot2

Why Use Geospatial Analysis?

üìå Geospatial analysis allows us to visualize the spatial distribution of malaria cases, identify hotspots, and understand geographic patterns that may not be apparent in tabular data.

Step 1: Setting Up the Environment


# Install required packages if not already installed

Step 2: Creating a Basic Map with Point Data

# Load sample malaria data with coordinates (lat/long) # In practice, you would load your actual data malaria_points <- read.csv("malaria_coordinates.csv") # Preview the data head(malaria_points) # Convert to sf object for spatial operations malaria_sf <- st_as_sf(malaria_points, coords = c("longitude", "latitude"), crs = 4326) # WGS84 coordinate system # Get country boundaries for background map countries <- ne_countries(scale = "medium", returnclass = "sf") # Create a basic map of malaria cases ggplot() + # Add country boundaries geom_sf(data = countries, fill = "lightgray", color = "white") + # Add malaria points geom_sf(data = malaria_sf, aes(color = cases, size = cases), alpha = 0.7) + # Color and size legends scale_color_viridis_c(name = "Malaria Cases") + scale_size_continuous(name = "Malaria Cases", range = c(1, 5)) + # Map extent - focus on a region, e.g., Africa coord_sf(xlim = c(-20, 50), ylim = c(-35, 35), expand = FALSE) + # Add labels and title labs( title = "Spatial Distribution of Malaria Cases", subtitle = "Point data showing case numbers by location", caption = "Data source: WHO Malaria Report" ) + theme_minimal()

üîç Explanation:

‚úî st_as_sf() converts regular data to spatial features
‚úî coords = c("longitude", "latitude") specifies which columns contain coordinates
‚úî crs = 4326 sets the coordinate reference system to WGS84 (standard for GPS data)
‚úî geom_sf() is ggplot2's function for plotting spatial features
‚úî We use color and size aesthetics to show the number of cases at each location

Step 4: Creating a Heatmap or Density Map

# Install the spatstat package for spatial statistics install.packages("spatstat") library(spatstat) library(raster) # Convert sf to spatial points for density analysis malaria_sp <- as(malaria_sf, "Spatial") malaria_ppp <- as(malaria_sp, "ppp") # Create a density map density_map <- density(malaria_ppp, sigma = 1) # Adjust sigma for smoothing # Convert to raster for plotting with ggplot2 density_raster <- raster(density_map) density_df <- as.data.frame(density_raster, xy = TRUE) # Plot the density map ggplot() + # Add country boundaries geom_sf(data = countries, fill = "white", color = "gray70") + # Add density raster geom_tile(data = density_df, aes(x = x, y = y, fill = layer)) + # Custom color scale for density scale_fill_viridis_c( name = "Case Density", option = "inferno", direction = -1, alpha = 0.8 ) + # Focus on a specific region coord_sf(xlim = c(-20, 50), ylim = c(-35, 35), expand = FALSE) + # Add informative labels labs( title = "Malaria Case Density Map", subtitle = "Hotspots of malaria transmission", caption = "Data source: WHO Malaria Report 2023" ) + theme_minimal()

üîç Explanation:

‚úî density() creates a kernel density estimate from point data
‚úî sigma controls the smoothing bandwidth (higher = smoother)
‚úî We convert between spatial data formats to leverage different packages' capabilities
‚úî geom_tile() plots the density as a continuous surface
‚úî This approach helps identify hotspots where cases are clustered

2Ô∏è‚É£ Reading Shapefiles

Step 1: Understanding Shapefiles

# Shapefile components explanation # .shp: geometric features # .dbf: attribute data # .shx: index file for shapes # .prj: projection information # Reading a shapefile with sf admin_boundaries <- st_read("admin_boundaries.shp") # Examine the shapefile structure print(admin_boundaries) head(admin_boundaries) # Plot the shapefile to verify it loaded correctly plot(st_geometry(admin_boundaries)) # Check the coordinate reference system (CRS) st_crs(admin_boundaries) # Transform to a different CRS if needed (e.g., to WGS84) admin_boundaries_wgs84 <- st_transform(admin_boundaries, 4326)

üîç Explanation:

‚úî Shapefiles are a common format for storing geographic data
‚úî st_read() loads shapefiles into R as sf objects
‚úî st_geometry() extracts just the geometric component
‚úî st_crs() checks the coordinate reference system
‚úî st_transform() converts between different coordinate systems
‚úî WGS84 (EPSG:4326) is a standard CRS for global mapping

Step 2: Examining Shapefile Attributes

# View attributes associated with shapefile names(admin_boundaries) # Examine a specific attribute table(admin_boundaries$ADMIN_LEVEL) # Basic statistics for a numeric attribute summary(admin_boundaries$POPULATION) # Select specific columns and filter regions malaria_regions <- admin_boundaries %>% select(REGION_NAME, POPULATION, AREA_KM2, geometry) %>% filter(REGION_NAME %in% c("Region1", "Region2", "Region3")) # Simplify complex geometries for faster plotting (if needed) admin_simplified <- st_simplify(admin_boundaries, dTolerance = 0.01) # Compare geometry complexity object.size(admin_boundaries) object.size(admin_simplified)

üîç Explanation:

‚úî Shapefiles contain both geometric data and attribute data
‚úî We can use dplyr functions like select() and filter() with sf objects
‚úî st_simplify() reduces the complexity of geometries
‚úî dTolerance controls how much simplification to apply
‚úî Simplifying geometries can significantly improve performance for complex maps

Step 3: Joining Malaria Data with Shapefiles

# Load tabular malaria data malaria_data <- read.csv("malaria_by_region.csv") # Inspect the malaria data head(malaria_data) # Join malaria data with shapefile # Make sure the joining column names match exactly malaria_map_data <- admin_boundaries %>% left_join(malaria_data, by = c("REGION_ID" = "region_id")) # Check if the join worked correctly head(malaria_map_data) # Verify that data is properly joined by plotting a sample attribute plot(malaria_map_data["malaria_cases"], main = "Malaria Cases by Region") # Calculate incidence rate per 1000 people malaria_map_data <- malaria_map_data %>% mutate( incidence_per_1000 = (malaria_cases / POPULATION) * 1000, # Create categories for mapping incidence_category = cut( incidence_per_1000, breaks = c(0, 1, 5, 10, 50, 100, Inf), labels = c("Very Low", "Low", "Moderate", "High", "Very High", "Extreme") ) )

üîç Explanation:

‚úî left_join() combines the shapefile with tabular malaria data
‚úî The by parameter specifies which columns to join on
‚úî mutate() calculates new variables like incidence rates
‚úî cut() creates categorical variables for easier mapping
‚úî Incidence rates standardize case counts by population size

Step 4: Working with Multiple Shapefiles

# Load multiple administrative levels admin1 <- st_read("admin1_boundaries.shp") # Provinces/states admin2 <- st_read("admin2_boundaries.shp") # Districts/counties # Load health facility locations health_facilities <- st_read("health_facilities.shp") # Create a layered map ggplot() + # Base layer: Admin1 boundaries geom_sf(data = admin1, fill = NA, color = "black", size = 1) + # Admin2 boundaries geom_sf(data = admin2, fill = NA, color = "gray", size = 0.5) + # Health facilities geom_sf(data = health_facilities, aes(color = FACILITY_TYPE), size = 3) + # Add labels for admin1 regions geom_sf_text(data = admin1, aes(label = NAME), size = 3, check_overlap = TRUE) + # Custom legend scale_color_brewer(name = "Facility Type", palette = "Set1") + # Add title and caption labs( title = "Health Facilities and Administrative Boundaries", subtitle = "Multiple shapefile layers combined", caption = "Data source: Ministry of Health" ) + theme_minimal() # Find which admin regions contain which facilities (spatial join) facilities_by_region <- st_join(health_facilities, admin1) # Summarize facilities by region facility_counts <- facilities_by_region %>% group_by(REGION_NAME) %>% summarize( total_facilities = n(), hospitals = sum(FACILITY_TYPE == "Hospital"), clinics = sum(FACILITY_TYPE == "Clinic"), .groups = "drop" ) print(facility_counts)

üîç Explanation:

‚úî Multiple shapefiles can be layered to create comprehensive maps
‚úî geom_sf_text() adds labels to areas on the map
‚úî st_join() performs a spatial join between different geometries
‚úî Spatial joins find which points fall within which polygons
‚úî We can then summarize data to count facilities by region

3Ô∏è‚É£ Creating Choropleth Maps

Step 1: Basic Choropleth Map with ggplot2

# Create a basic choropleth map of malaria incidence ggplot(data = malaria_map_data) + # Fill regions by incidence rate geom_sf(aes(fill = incidence_per_1000)) + # Add borders in white geom_sf(fill = NA, color = "white", size = 0.2) + # Custom color scale scale_fill_viridis_c( name = "Incidence per 1000", option = "plasma", direction = -1, trans = "sqrt" # Square root transform to better show differences ) + # Add title and labels labs( title = "Malaria Incidence Rate by Region", subtitle = "Cases per 1000 population", caption = "Data source: Ministry of Health, 2023" ) + # Minimal theme for maps theme_minimal() + theme( legend.position = "right", plot.title = element_text(size = 16, face = "bold"), axis.text = element_text(size = 8) )

üîç Explanation:

‚úî A choropleth map uses color to represent data values across regions
‚úî geom_sf(aes(fill = ...)) colors regions based on data values
‚úî scale_fill_viridis_c() uses colorblind-friendly color schemes
‚úî trans = "sqrt" applies a square root transformation to highlight differences in lower values
‚úî White borders help distinguish between adjacent regions

Step 2: Advanced Choropleth Maps with tmap

# Install and load the tmap package install.packages("tmap") library(tmap) # Set tmap mode to plot (static) or view (interactive) tmap_mode("plot") # For static maps # tmap_mode("view") # For interactive maps # Create a basic choropleth map with tmap tm_shape(malaria_map_data) + tm_fill("incidence_per_1000", title = "Incidence per 1000", palette = "YlOrRd", style = "jenks", # Natural breaks classification n = 6) + # Number of classes tm_borders(col = "white", lwd = 0.5) + tm_layout( title = "Malaria Incidence Rate Map", title.position = c("center", "top"), legend.outside = TRUE, legend.outside.position = "right", frame = FALSE ) # Create a more complex map with multiple variables tm_shape(malaria_map_data) + tm_fill("incidence_category", title = "Incidence Level", palette = "YlOrRd", alpha = 0.7) + tm_borders(col = "white", lwd = 0.5) + # Add circle markers showing total cases tm_shape(st_centroid(malaria_map_data)) + tm_bubbles( size = "malaria_cases", col = "blue", alpha = 0.6, scale = 2, title.size = "Total Cases", id = "REGION_NAME" # For interactive tooltips ) + # Add text labels for high-incidence areas tm_text("REGION_NAME", size = 0.7, col = "black", just = "center", fontface = "bold", auto.placement = TRUE, remove.overlap = TRUE) + # Layout and styling tm_layout( title = "Malaria Burden by Region", title.size = 1.5, legend.outside = TRUE, legend.outside.position = "right", frame = FALSE ) + tm_compass(position = c("left", "bottom")) + tm_scale_bar(position = c("left", "bottom"))

üîç Explanation:

‚úî tmap is specialized for thematic mapping with a layered approach
‚úî tm_shape() defines the spatial data to be mapped
‚úî tm_fill() creates the choropleth coloring
‚úî style = "jenks" uses natural breaks to classify the data
‚úî tm_bubbles() adds proportional symbols on the map
‚úî tmap_mode("view") creates interactive maps with zoom and pan capabilities

Step 3: Creating a Map Series with Faceting

# Assume we have malaria data for multiple years malaria_multi_year <- read.csv("malaria_by_region_years.csv") # Join with shapefile malaria_map_timeseries <- admin_boundaries %>% left_join(malaria_multi_year, by = c("REGION_ID" = "region_id")) # Using ggplot2 for faceted maps ggplot(data = malaria_map_timeseries) + geom_sf(aes(fill = incidence_per_1000)) + geom_sf(fill = NA, color = "white", size = 0.1) + scale_fill_viridis_c(option = "plasma", direction = -1, name = "Incidence per 1000") + facet_wrap(~year, ncol = 2) + labs( title = "Malaria Incidence Over Time", subtitle = "Changes in spatial distribution (2018-2022)", caption = "Data source: Ministry of Health" ) + theme_minimal() + theme( axis.text = element_blank(), legend.position = "right" ) # Using tmap for faceted maps tm_shape(malaria_map_timeseries) + tm_fill("incidence_per_1000", palette = "YlOrRd", style = "fisher", title = "Incidence per 1000") + tm_borders(col = "white", lwd = 0.2) + tm_facets(by = "year", ncol = 2, free.scales = FALSE) + tm_layout( title = "Malaria Incidence (2018-2022)", legend.outside = TRUE, panel.labels = c("2018", "2019", "2020", "2021", "2022"), panel.label.size = 1.2, frame = FALSE )

üîç Explanation:

‚úî Faceted maps show how spatial patterns change over time or across categories
‚úî facet_wrap(~year) in ggplot2 creates a series of maps by year
‚úî tm_facets() in tmap provides similar functionality
‚úî free.scales = FALSE ensures the same color scale is used across all maps
‚úî This approach helps visualize temporal trends in spatial distribution

Step 5: Interactive Maps with tmap and leaflet
